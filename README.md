# CompilePro
现代智能项目编译管理平台
## 目标
整合平台部署时，涉及到的所有项目，提炼公共配置，减少运维对配置理解的门槛和要求
优化版本管理，解决现在版本管理混乱不清晰的情况，使每个人，都能够清楚了解用户部署的是哪个版本，这个版本都有哪些内容，可以升级到哪个版本，版本之间迭代，迭代了什么内容
让每一次的编译部署都有迹可循
## 整体思路
提炼出来三个概念：git源、模板、项目
git源：即每个单独的git库，平台中，会把每个git项目中，需要暴露的配置项提取出来
模板：即部署模板，由不同的git源组成，一个模板应该为一个类型的部署，比如私有部署标准版、私有部署单机版、线上sass版、测试版等
项目：选择指定的部署模板，以不同的环境、客户为单位，一个客户部署，即为一个项目。
